# 变更提案: 登录页布局与动画复刻

## 需求背景

当前需要将 Vue 版登录页的布局与动画迁移到 Next.js 页面，以保证登录/注册切换体验一致，并为后续图片资源接入预留位置。

## 产品分析

### 目标用户与场景

- **用户群体:** 首次访问或回访的普通用户
- **使用场景:** 登录/注册页切换与表单填写
- **核心痛点:** 动效缺失导致切换体验生硬

### 价值主张与成功指标

- **价值主张:** 保持跨技术栈一致的视觉与交互体验
- **成功指标:** 登录/注册切换动效与布局和 Vue 版一致，图片可后续替换

### 人文关怀

动效与文字提示清晰可读，避免视觉干扰。

## 变更内容

1. 重写 `src/app/login/page.tsx`，复刻 Vue 登录页布局与切换动画。
2. 引用现有 React 版 `LoginForm` 与 `RegisterForm`。
3. 预留图片容器，暂不接入图片资源。

## 影响范围

- **模块:** 登录页、通用业务组件
- **文件:** `src/app/login/page.tsx`
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: 登录/注册切换动效

**模块:** 登录页用户可在登录与注册模式之间切换。

#### 场景: 切换到注册模式

用户点击“去注册”按钮。

- 登录表单移出，注册表单移入
- 面板文字与占位图片区切换动效生效

#### 场景: 切换到登录模式

用户点击“去登录”按钮。

- 注册表单移出，登录表单移入
- 面板文字与占位图片区切换动效生效

## 风险评估

- **风险:** 动画与布局细节在响应式场景下表现偏差
- **缓解:** 按 Vue 版断点与动效参数实现
