# 变更提案: 登录与注册表单组件

## 需求背景

当前需要将既有 Vue 版本的登录与注册表单迁移到 React（Next.js）版本，以保持页面结构、样式与交互一致，并为后续接口对接预留位置。

## 产品分析

### 目标用户与场景

- **用户群体:** 首次访问或回访的普通用户
- **使用场景:** 访问登录/注册页完成账号认证
- **核心痛点:** 表单交互不一致导致学习成本增加

### 价值主张与成功指标

- **价值主张:** 在新技术栈中保持一致体验与操作路径
- **成功指标:** 登录/注册表单可用，校验提示与跳转行为一致

### 人文关怀

避免过度错误提示暴露敏感信息，保持提示友好清晰。

## 变更内容

1. 实现 React 版登录表单组件，覆盖输入校验、密码可见切换与成功跳转。
2. 实现 React 版注册表单组件，覆盖确认密码校验、密码可见切换与成功跳转。
3. 调整登录页引用，使用更合适的组件命名。

## 影响范围

- **模块:** UI 通用业务组件、登录页
- **文件:** `src/components/common/LoginForm.tsx`,
  `src/components/common/RegisterForm.tsx`, `src/app/login/page.tsx`
- **API:** 无（预留位置）
- **数据:** 无

## 核心场景

### 需求: 登录表单

**模块:** 通用业务组件实现登录表单的输入、校验与跳转。

#### 场景: 正常登录

用户输入用户名和密码后提交。

- 表单校验通过后模拟成功并跳转 `/explore`

### 需求: 注册表单

**模块:** 通用业务组件实现注册表单的输入、校验与跳转。

#### 场景: 注册成功跳转

用户输入用户名、密码、确认密码并提交。

- 密码一致校验通过后模拟成功并跳转 `/explore`

## 风险评估

- **风险:** 未对接真实接口导致后续集成时需补充逻辑
- **缓解:** 预留 API 调用位置并清晰标注
