# tooling

## 目的

统一工程化工具链与脚手架配置，确保多人协作下的开发体验与可维护性。

## 模块概述

- **职责:** 工程化工具链、脚手架、包管理与提交规范
- **状态:** ✅稳定
- **最后更新:** 2026-01-19

## 规范

### 需求: 工程化工具链

**模块:** tooling通过统一工具链与配置降低协作成本，保证一致性与可回溯性。

#### 场景: 脚手架初始化

前置条件：项目需基于 Next.js + TypeScript + App Router 初始化。

- 统一基础目录结构与依赖版本
- 保持基础 UI/样式框架可扩展

#### 场景: 提交规范与格式化

前置条件：多人协作频繁提交。

- commitlint + cz-git + Husky 保证提交规范
- Prettier 与 Tailwind 类名排序保持一致风格

#### 场景: Git 忽略策略

前置条件：存在构建产物、缓存与本地环境文件。

- .gitignore 覆盖构建产物、测试报告、日志、缓存与本地工具目录
- 以可重建的输出为主，避免提交本地机器状态文件
