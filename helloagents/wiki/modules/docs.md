# docs

## 目的

沉淀前端工程化相关的规范、建议与最佳实践，作为团队共识与落地参考。

## 模块概述

- **职责:** 文档管理、工程化实践沉淀、后续与代码实现对齐
- **状态:** ✅稳定
- **最后更新:** 2026-01-19

## 规范

### 需求: 工程化知识沉淀

**模块:**
docs在本仓库中记录与维护工程化规范与最佳实践，确保可读、可复用、可追溯。

#### 场景: 阅读与落地

前置条件：仓库包含工程化相关文档。

- 提供清晰的主题划分与索引，便于检索与复用
- 当代码与文档不一致时，以代码为准并更新文档（SSOT 需保持最新）

#### 场景: 文件用途清单

前置条件：需要快速理解仓库文件结构与用途。

- doc/ 内提供文件用途清单，按路径列出所有文件职责
- 文档更新后同步知识库变更历史
