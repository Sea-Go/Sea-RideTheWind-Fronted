# Sea-TryGo 帖子平台前端页面细分设计文档

> 版本：v1.0  
> 日期：2026-03-13  
> 依据文档：`doc/接口文档/接口说明-六大业务模块.md`（31 个接口）

## 1. 目标与范围

本文档用于定义“帖子平台”前端在功能闭环上需要的页面、交互和接口映射，覆盖：

- 用户侧：发帖、浏览帖子、点赞、评论查看、个性化推荐、个人信息、任务进度
- 管理侧：管理员登录、用户管理、账号安全管理
- 技术侧：BFF 同源调用策略、接口封装调用点、页面权限控制

## 2. 产品定位与核心场景

平台核心目标：

- 用户可发布帖子并参与互动（点赞、评论浏览）
- 系统根据用户行为偏好推荐帖子，形成个性化首页

核心用户旅程：

1. 注册/登录
2. 进入推荐首页浏览帖子流
3. 打开帖子详情并互动（点赞、查看评论）
4. 发布/编辑自己的帖子
5. 查看个人获赞与任务进度

## 3. 信息架构（IA）

### 3.1 用户端（Web）

- `/login` 登录注册页
- `/` 推荐首页（个性化 feed）
- `/post/[id]` 帖子详情页
- `/editor/new` 发帖页
- `/editor/[id]` 编辑帖子页
- `/me` 个人中心页
- `/me/posts` 我的帖子管理页
- `/me/likes` 我的点赞与获赞页
- `/me/tasks` 任务进度页

### 3.2 管理端（Web）

- `/admin/login` 管理员登录页
- `/admin` 管理工作台
- `/admin/users` 用户列表页
- `/admin/users/[uid]` 用户详情与操作页
- `/admin/profile` 管理员个人信息页

## 4. 页面清单与接口映射

## 4.1 用户端页面

### A. 登录注册页 `/login`

功能：

- 用户注册
- 用户登录并存储 token
- 登录后跳转推荐首页

接口：

- `POST /usercenter/v1/user/register`
- `POST /usercenter/v1/user/login`

前端封装建议：

- `services/auth.ts`：`registerUser`、`loginUser`

UI 重点：

- 左右结构（品牌介绍 + 表单）
- 注册/登录切换 Tab
- 密码可见性切换、提交 loading、错误提示

---

### B. 推荐首页 `/`

功能：

- 基于用户身份获取推荐帖子
- 展示帖子卡片流
- 点赞、快速查看点赞状态/点赞数
- 支持按标签或时间排序过滤（可选）

接口：

- `POST /api/v1/reco/recommend`
- `GET /v1/articles`
- `POST /like/v1/getlikestate`
- `POST /like/v1/getlikecount`
- `POST /like/v1/likeaction`

前端封装建议：

- `services/reco.ts`：`recommendArticles`
- `services/article.ts`：`listArticles`
- `services/like.ts`：`getLikeState`、`getLikeCount`、`likeAction`

UI 重点：

- 顶部搜索与筛选条（排序、标签）
- Feed 卡片（封面、标题、摘要、作者、互动数）
- 点赞按钮支持乐观更新

---

### C. 帖子详情页 `/post/[id]`

功能：

- 展示单篇帖子完整内容
- 点赞/取消点赞
- 查看评论列表与楼中楼结构
- 展示相关推荐（可选）

接口：

- `GET /v1/article/{id}`
- `POST /like/v1/likeaction`
- `POST /like/v1/getlikecount`
- `POST /like/v1/getlikestate`
- `POST /comment/v1/comment/list`
- `POST /api/v1/reco/recommend`（可选：相关推荐）

前端封装建议：

- `services/article.ts`：`getArticle`
- `services/like.ts`：`likeAction`、`getLikeCount`、`getLikeState`
- `services/comment.ts`：`getCommentList`

UI 重点：

- 内容区 + 右侧信息区布局（桌面）
- 评论区支持分页/“加载更多”
- 评论输入框可先占位（当前无“新增评论”接口）

---

### D. 发帖页 `/editor/new`

功能：

- 输入标题、摘要、正文、封面、分类标签
- 提交创建帖子

接口：

- `POST /v1/article`

前端封装建议：

- `services/article.ts`：`createArticle`

UI 重点：

- 富文本编辑器 + 实时预览
- 封面上传/选择
- 草稿态（本地缓存）

---

### E. 编辑帖子页 `/editor/[id]`

功能：

- 拉取帖子并回填
- 更新帖子
- 删除帖子

接口：

- `GET /v1/article/{id}`
- `PUT /v1/article/{id}`
- `DELETE /v1/article/{id}`

前端封装建议：

- `services/article.ts`：`getArticle`、`updateArticle`、`deleteArticle`

UI 重点：

- 编辑区与发帖页保持一致
- 删除需二次确认
- 更新成功后返回详情页

---

### F. 个人中心页 `/me`

功能：

- 展示用户资料、积分、偏好信息
- 编辑资料
- 退出登录
- 注销账号

接口：

- `GET /usercenter/v1/user/get`
- `POST /usercenter/v1/user/update`
- `POST /usercenter/v1/user/logout`
- `DELETE /usercenter/v1/user/delete`

前端封装建议：

- `services/auth.ts`：`getUserProfile`、`updateUserProfile`、`logoutUser`、`deleteCurrentUser`

UI 重点：

- 资料卡 + 编辑抽屉
- 危险操作区（注销）单独视觉强调

---

### G. 我的帖子管理页 `/me/posts`

功能：

- 展示我发布的帖子
- 进入编辑、删除

接口：

- `GET /v1/articles?author_id=<me>`
- `DELETE /v1/article/{id}`

前端封装建议：

- `services/article.ts`：`listArticles`、`deleteArticle`

UI 重点：

- 表格/卡片双视图
- 批量操作预留（后续）

---

### H. 我的点赞与获赞页 `/me/likes`

功能：

- 我点赞过的内容列表
- 我的累计获赞数

接口：

- `POST /like/v1/getuserlikelist`
- `POST /like/v1/getusertotallike`
- `POST /like/v1/gettargetlikerlist`（用于某一内容的点赞用户列表）

前端封装建议：

- `services/like.ts`：`getUserLikeList`、`getUserTotalLike`、`getTargetLikerList`

UI 重点：

- 统计卡片 + 列表区域
- 游标分页滚动加载

---

### I. 任务进度页 `/me/tasks`

功能：

- 展示任务列表及完成进度

前端封装建议：

## 6. UI 设计规范建议（适配帖子平台）

视觉风格建议：

- 信息密度中高，强调内容阅读
- 首页卡片采用 12 栅格，桌面 3 列 / 平板 2 列 / 手机 1 列
- 主色建议蓝绿系（信任 + 活力），互动色（点赞）使用高对比暖色

关键组件：

- `PostCard`：封面、标题、摘要、作者、点赞/评论数
- `LikeButton`：三态（未点赞/已点赞/处理中）
- `CommentPanel`：主评论 + 子评论
- `TaskProgressCard`：任务标题、描述、进度条、完成态
- `ProfileEditor`：资料编辑抽屉
- `AdminUserActionBar`：封禁/解封/重置/删除快捷操作

状态设计：

- 空态：无内容时给出明确 CTA（去发帖、去浏览）
- 加载态：骨架屏优先，避免闪烁
- 错误态：可重试，显示可理解错误文案

## 7. 权限与路由守卫

用户端：

- 未登录可访问：`/`、`/post/[id]`、`/login`
- 需登录：`/editor/*`、`/me/*`

管理端：

- 仅管理员 token 可访问 `/admin/*`
- 普通用户访问 `/admin/*` 直接重定向登录页

## 9. 风险与备注

- 当前 comment 模块仅有“评论列表”接口，无“新增评论”接口。
  - 前端策略：评论输入先做 UI 占位，提交按钮置灰并提示“功能待开放”。
- Article 接口在 OAS 中存在固定 ID 示例路径。
  - 前端策略：统一按 `/v1/article/{id}` 动态路由封装。
- 部分接口返回 schema 未完全结构化。
  - 前端策略：联调阶段先落 `Record<string, unknown>`，稳定后再收敛为严格类型。
