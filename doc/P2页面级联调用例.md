# P2 页面级联调用例

> 日期：2026-03-15  
> 目标页面：`/profile/likes`、`/profile/tasks`、`/profile/security`

## 用例 1（成功）个人中心扩展页联动

### 前置条件
- 用户已登录，浏览器已存在 `user_center_token`（localStorage/cookie 任一）。
- `LIKE_API_SERVER_URL`、`TASK_CENTER_API_SERVER_URL`、`USER_CENTER_API_SERVER_URL` 可访问。

### 步骤
1. 访问 `/profile`，点击“我的点赞”进入 `/profile/likes`。
2. 页面调用：
   - `POST /api/like/v1/getusertotallike`
   - `POST /api/like/v1/getuserlikelist`
3. 点击“加载更多”，继续调用 `getuserlikelist`（cursor 分页）。
4. 返回 `/profile`，点击“任务进度”进入 `/profile/tasks`。
5. 页面调用 `POST /api/taskcenter/v1/task/get`，显示进行中/已完成分组。
6. 返回 `/profile`，点击“安全设置”进入 `/profile/security`，点击“退出登录”。

### 预期结果
- 三个页面均可访问并正常显示加载态、空态或数据态。
- 点赞页统计与列表可展示，分页按钮行为正常。
- 任务页可刷新并按状态分组。
- 退出登录后清理 token 并回到 `/login`。

## 用例 2（失败）受保护路由与接口异常兜底

### 前置条件
- 用户未登录，或手动清除 token。
- 可选：模拟 `task/get` 或 `getuserlikelist` 接口失败。

### 步骤
1. 直接访问 `/profile/likes`、`/profile/tasks`、`/profile/security`。
2. 观察中间件重定向到 `/login?next=...`。
3. 登录后自动回跳目标页。
4. 在目标页触发接口失败场景（如后端不可用）。

### 预期结果
- 未登录访问受保护路由时不会暴露页面数据。
- 登录后可按 `next` 参数安全回跳。
- 接口失败时页面展示错误提示，且提供重试能力（刷新/加载更多/按钮重试）。
