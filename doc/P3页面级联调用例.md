# P3 页面级联调用例

> 日期：2026-03-15  
> 目标页面：`/admin/login`、`/admin`、`/admin/users`、`/admin/users/[uid]`、`/admin/profile`

## 用例 1（成功）管理端主链路

### 前置条件
- 管理员账号可用。
- `ADMIN_CENTER_API_SERVER_URL` 可访问。

### 步骤
1. 访问 `/admin/users`，中间件重定向到 `/admin/login?next=/admin/users`。
2. 在 `/admin/login` 输入管理员账号密码，登录成功回跳 `/admin/users`。
3. 用户列表页执行检索与分页，调用 `GET /api/admincenter/v1/admin/getuserlist`。
4. 进入某用户详情 `/admin/users/{uid}`，调用 `GET /api/admincenter/v1/admin/getuser`。
5. 在详情页依次执行“更新用户”“封禁/解封”“重置密码”操作。
6. 返回 `/admin/profile`，执行管理员资料更新与新管理员创建。

### 预期结果
- 管理端登录态可持续，页面可在 `/admin/*` 间流转。
- 用户详情操作后有成功提示，并刷新用户最新状态。
- 管理员资料更新、新建管理员操作可正常反馈结果。

## 用例 2（失败）鉴权与高风险操作兜底

### 前置条件
- 清空 `admin_center_token`。
- 可选：模拟后端接口返回异常。

### 步骤
1. 直接访问 `/admin`、`/admin/profile`、`/admin/users/{uid}`。
2. 验证均跳转到 `/admin/login?next=...`。
3. 登录后在用户详情页执行“删除用户”，点击取消确认。
4. 再次执行删除并让接口失败（如后端不可用）。

### 预期结果
- 未登录时不会暴露管理页面内容。
- 取消确认后不发起删除请求。
- 删除失败时页面显示错误信息，不发生错误跳转。
