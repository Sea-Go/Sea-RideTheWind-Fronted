# AI 文献摘要与封面生成产品需求文档（PRD）

## 1. 背景与问题

在学术阅读、内容创作与知识管理场景中，用户经常需要快速理解一篇文献的核心内容，并为其生成一个可用于展示、分享或发布的平台封面图。目前存在的问题是：

- 文献篇幅长，核心信息分散，人工总结成本高
- 生成封面时，设计师/作者需要重新理解内容，重复劳动
- 现有 AI 工具多为“一步到位”，结果不可控、不可编辑、不可复用

## 2. 产品目标

通过 AI 能力，实现：

1. 自动概括文献的**主要内容与核心贡献**
2. 基于结构化摘要，自动生成**符合内容语义的封面图片**
3. 提供**可解释、可编辑、可复用**的中间结果，支持产品化使用

## 3. 用户与使用场景

### 3.1 目标用户

- 学生 / 研究人员（论文阅读、汇报材料）
- 内容创作者（技术博客、课程资料）
- 内部研发 / 知识库维护人员

### 3.2 使用场景

- 上传 PDF / 文本 → 快速理解论文内容
- 为文章 / 文献生成默认封面
- 在发布前手动微调摘要或封面风格

## 4. 核心功能需求

### 4.1 文献输入

- 支持格式：PDF、Markdown、纯文本
- 支持粘贴文本或文件上传
- 限制单次最大字数（如 50k tokens，超出走分段处理）

### 4.2 文献摘要生成（核心）

AI 对文献进行理解，输出**结构化摘要**，而非纯自然语言总结。

#### 摘要内容字段（V1）

- 文献标题
- 研究领域 / 主题
- 核心问题（解决了什么）
- 关键贡献（1~3 条）
- 关键词（3~6 个）
- 整体语气 / 风格（学术 / 工程 / 前沿等）

要求：
- 基于原文，不允许编造
- 不确定内容需标注 unknown

### 4.3 摘要可编辑

- 用户可手动编辑摘要字段
- 编辑后作为“最终摘要”进入下一步

### 4.4 封面生成

- 基于最终摘要生成封面图
- 默认生成 1 张
- 支持重新生成（同摘要，不同风格）

#### 封面约束（V1）

- 默认比例：3:4
- 无文字、无水印
- 风格偏学术 / 简洁 / 抽象

### 4.5 结果输出

- 展示结构化摘要
- 展示生成的封面图片
- 支持下载封面

## 5. 非功能性需求

- 响应时间：
  - 摘要生成 ≤ 10s（长文档除外）
  - 封面生成 ≤ 15s
- 可扩展性：支持后续增加多风格、多语言
- 可维护性：摘要与封面生成逻辑解耦

## 6. 不在本期范围（V1 不做）

- 多文献对比
- 人工协作编辑
- 封面中文字自动排版

## 7. 验收标准（示例）

- 输入一篇 5k 字以上论文，可生成结构化摘要
- 摘要字段齐全，语义与原文一致
- 基于摘要成功生成封面图片
- 修改关键词后，封面视觉明显变化
