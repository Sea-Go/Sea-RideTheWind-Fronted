# AI 封面生成技术方案

## 1. 背景与目标

在用户社区产品中（类似小红书 / 掘金），帖子封面直接影响内容点击率与整体内容质量。但在实际使用中，部分用户在发布文章时不会主动上传封面，导致内容展示效果参差不齐。

为提升内容发布成功率与内容质量下限，同时不增加用户操作负担，引入
**AI 自动生成封面** 作为兜底能力。

### 目标

- 保证 **每篇文章都有封面**
- 不阻塞用户发布主流程
- AI 失败不影响内容发布
- 封面生成过程可控、可降级、可替换

---

## 2. 设计原则

1. **AI 非强依赖**：AI 封面仅作为兜底方案，不作为发布前置条件
2. **异步生成**：避免阻塞发布流程
3. **前后端职责清晰**：前端不直接调用 AI 服务
4. **模型可替换**：支持未来切换不同文生图模型
5. **合规优先**：避免生成高风险内容

---

## 3. 整体架构设计

```text
用户发布文章
    ↓
前端校验是否上传封面
    ↓（无封面）
后端创建文章（绑定默认封面）
    ↓
异步触发 AI 封面生成任务
    ↓
AI 文生图服务
    ↓
图片上传至对象存储（OSS / CDN）
    ↓
更新文章封面信息
```

### 架构说明

- 发布接口 **同步返回成功**，不等待 AI 结果
- AI 封面生成通过异步任务或消息队列执行
- 生成失败不会回滚文章数据

---

## 4. 前端方案设计（React Web）

### 4.1 前端职责

- 校验是否上传封面
- 提交文章内容
- 展示封面生成状态
- 接收封面更新结果
- 允许用户手动替换封面

### 4.2 发布流程状态设计

```ts
enum PublishStatus {
  Idle,
  Submitting,
  Published,
  CoverGenerating,
  Completed,
  Error,
}
```

### 4.3 用户体验设计

- 发布成功后立即进入文章页
- 若封面为 AI 生成：
  - 显示「AI 正在生成封面」提示
  - 生成完成后自动刷新封面
  - 提示用户："该封面由 AI 自动生成，可随时替换"

---

## 5. 接口设计

### 5.1 发布文章接口

```http
POST /api/posts
```

```json
{
  "title": "深入理解 React Fiber",
  "content": "<p>...</p>",
  "cover": null
}
```

### 5.2 封面状态查询接口（轮询方案）

```http
GET /api/posts/{postId}/cover-status
```

```json
{
  "status": "generated",
  "coverUrl": "https://cdn.xxx/cover.png",
  "isAIGenerated": true
}
```

> 说明：
>
> - `status`: pending | generated | failed
> - 前端可在文章发布后进行有限次数轮询

## 6. AI 封面生成服务设计

### 6.1 输入数据

- title（文章标题）
- summary（文章摘要，≤200 字）
- tags（可选）

> summary 可由后端统一生成，避免前端传全文

### 6.2 Prompt 设计原则

- 风格：插画 / 抽象 / 场景
- 禁止元素：
  - 人脸
  - 文字
  - 水印
  - 政治 / 暴力 / 敏感内容

### Prompt 示例

```text
Create a clean, modern cover illustration for a tech community article.
Topic: React frontend development
Style: minimal, flat illustration
Color: soft gradient
No text, no watermark, no human face
Aspect ratio: 16:9
```

---

## 7. 模型选型建议

| 方案              | 优点             | 备注             |
| ----------------- | ---------------- | ---------------- |
| Stable Diffusion  | 成本低、可控性强 | 推荐自部署或 API |
| OpenAI Images API | 稳定、生态成熟   | 成本较高         |
| 国内大模型        | 中文理解好       | 风格一致性需评估 |

> 模型调用统一封装在 AI Cover Service 中，前端无感知

---

## 8. 并发、限流与缓存策略

### 并发控制

- 单篇文章仅允许生成一次
- 相同内容重复发布可命中缓存

### 缓存策略

```text
hash(title + summary) → coverUrl
```

- 可使用 Redis / KV 存储
- 命中缓存直接复用封面

---

## 9. 降级与兜底方案

| 场景          | 行为             |
| ------------- | ---------------- |
| AI 超时       | 使用默认封面     |
| AI 生成失败   | 记录日志，不重试 |
| AI 服务不可用 | 完全跳过生成     |

> 默认封面可按分类 / 标签区分

---

## 10. 合规与风控设计

1. **UI 标识**：AI 封面需标注“AI 自动生成”
2. **用户可控**：允许用户随时替换封面
3. **内容限制**：Prompt 层强约束生成范围
4. **审计能力**：记录生成日志与 Prompt 版本

---

## 11. 未来扩展方向

- 支持用户手动触发“重新生成封面”
- 多封面候选（A/B 选择）
- 按内容分类定制封面风格
- 数据分析：AI 封面对点击率的影响

---

## 12. 总结

该方案通过 **异步 AI 封面生成 + 默认兜底封面**
的方式，在不影响主发布流程的前提下，提升内容展示质量。

整体设计具备：

- 可维护性
- 可扩展性
- 可控风险

适用于正式团队项目与长期演进。
