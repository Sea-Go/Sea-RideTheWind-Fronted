# P1 页面级联调用例（收尾）

> 日期：2026-03-15  
> 目标：覆盖 P1 主链路 1 条成功路径 + 1 条失败路径

## 用例 1（成功）登录后发布并进入详情

### 前置条件
- 用户账号存在且可正常登录。
- `USER_CENTER_API_SERVER_URL`、`ARTICLE_API_SERVER_URL`、`LIKE_API_SERVER_URL`、`COMMENT_API_SERVER_URL` 可访问。

### 步骤
1. 访问 `/post`，被中间件重定向到 `/login?next=/post`。
2. 在 `/login` 输入正确账号密码，登录成功后自动回跳 `/post`。
3. 在发布页输入标题、摘要、正文；上传封面或点击“AI 生成封面”。
4. 点击“发布”，前端调用 `POST /api/article/v1/article`。
5. 发布成功后跳转到 `/article/{id}`（若响应无 id，则回 `/dashboard/recommend`）。
6. 在详情页点击点赞，调用 `POST /api/like/v1/likeaction`。
7. 在详情页查看评论列表，调用 `POST /api/comment/v1/comment/list`。

### 预期结果
- 页面链路完整无断链：`/login -> /post -> /article/{id}`。
- 发布成功后可在详情页看到文章内容与互动区。
- 点赞按钮状态变更，评论区可正常显示加载态/空态。

## 用例 2（失败）发布失败回滚与提示

### 前置条件
- 用户已登录，能访问 `/post`。
- 人为制造失败条件：关闭 `ARTICLE_API_SERVER_URL` 或让发布接口返回错误。

### 步骤
1. 在 `/post` 填写完整内容并点击“发布”。
2. 前端调用 `POST /api/article/v1/article` 失败。
3. 观察发布页状态与提示。
4. 修复后端可用性后再次点击“发布”。

### 预期结果
- 失败时页面不跳转，展示错误文案（来源于服务端 `msg` 或前端兜底）。
- 发布按钮 loading 正常结束，可再次尝试发布。
- 表单内容保留，不丢失用户输入。
